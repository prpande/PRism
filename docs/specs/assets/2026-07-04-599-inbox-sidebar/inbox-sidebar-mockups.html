Per-section counts already exist (the pill by each title).
Sections already collapse — but all four default open, so a busy inbox is tall.
The pain: reaching "Awaiting author" and "Mentioned" means scrolling up and down past everything above.

Direction: sections become a collapsible left sidebar

Master-detail. Pick a section on the left; only that section's PRs fill the main pane. No more scrolling past three sections to reach the fourth. The sidebar keeps every section's count visible, so the at-a-glance triage survives.

Inside a busy section: cap the scroll by collapsing lower-order repos

Under the cap? Every repo stays open — identical to today. The rule only bites on genuinely busy multi-repo sections.
Top-of-sort first comes free from the existing repo order (backend emits in sort order; cold open = freshest).
The gap: if ONE repo holds all 24, there's nothing to collapse — accordions are all-or-nothing. Accepted; row-level "show top N" is a follow-up.
