# v2 AI P1b-1 — Base-Rebase Freshness Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close R2 (base-rebase staleness) by re-keying the in-memory PR-summary cache on `(prRef, baseSha, headSha)`, adding a live base-change producer (poll → event → SSE) that evicts the stale summary and surfaces a user-triggered "Out of date" badge + Regenerate control on the summary card.

**Architecture:** A base move on a same-head PR changes the diff (`base..head`) but not P1a's head-only cache key, so a stale summary is served. This slice teaches the GitHub cheap-poll to read `base.sha`, threads it through `ActivePrPollSnapshot` → `ActivePrPoller` (a new `baseChanged` publish-gate arm) → `ActivePrUpdated` (additive `BaseShaChanged`/`NewBaseSha`) → the SSE `pr-updated` frame. Two synchronous bus subscribers evict on a base move: `ClaudeCodeSummarizer` (its summary entries) and `PrDetailLoader` (its detail snapshot, so the fresh `baseSha` re-fetches). The summary cache key becomes a `SummaryCacheKey(PrRef, BaseSha, HeadSha)` record struct, and writes are guarded by an R7 SHA compare-and-set against the active-PR snapshot. The frontend latches `baseShaChanged` off SSE, drives `isStale`, and renders a Live-only stale chip + Regenerate that POSTs a new gated regenerate endpoint (one deliberate re-spend; no auto-spend).

**Tech Stack:** .NET 10 / C# 14 (xUnit + FluentAssertions, `PRismWebApplicationFactory`), React + Vite + TypeScript strict (Vitest + RTL), Playwright e2e with Linux visual baselines.

**Spec:** `docs/specs/2026-06-12-v2-ai-p1b-design.md`. **Tracker:** #374 (P0-2 base-sha cache-key child). **Branch / Base:** `feat/v2-ai-p1b` → `V2` (never `main`). **Worktree:** `C:\src\PRism-v2-p1b`.

If they momentarily disagree for a live PR (transient REST/GraphQL replication lag), the guard `current.BaseSha == baseSha` is false and a **valid** just-computed summary is silently dropped → a redundant provider call on the next GET. **Worst case is a bounded, self-healing redundant re-spend — never a stale serve and never a wrong answer** (the user always gets a correct summary). Task 0's spike conclusion (REST `base.sha` ≡ GraphQL `baseRefOid` for a live PR at steady state) is therefore **load-bearing for R7's read side too**, not only for the producer. Two mitigations, in order of preference: (a) if Task 0 selects the GraphQL fallback for the poller, the two sources **converge** and this risk disappears — prefer this if the spike is at all ambiguous; (b) otherwise accept the bounded re-spend and rely on the `After_regenerate_a_subsequent_GET_is_a_cache_HIT` test (Task 10) to catch any *systematic* (non-transient) false-reject. A full elimination — R7 reading the loader's GraphQL base instead of the poller's REST base — is possible but the summarizer does not hold the loader; defer that hardening unless the spike is inconclusive.

```csharp
public static IEndpointRouteBuilder MapAi(this IEndpointRouteBuilder app)
{
    ArgumentNullException.ThrowIfNull(app);

    app.MapGet("/api/pr/{owner}/{repo}/{number:int}/ai/summary",
        (string owner, string repo, int number, IAiSeamSelector ai, IActivePrCache activePrCache, CancellationToken ct) =>
            ResolveSummaryAsync(new PrReference(owner, repo, number), ai, activePrCache, regenerate: false, ct));

    // Deliberate re-spend (state-changing) → POST, CSRF-covered by OriginCheckMiddleware.
    app.MapPost("/api/pr/{owner}/{repo}/{number:int}/ai/summary/regenerate",
        (string owner, string repo, int number, IAiSeamSelector ai, IActivePrCache activePrCache, CancellationToken ct) =>
            ResolveSummaryAsync(new PrReference(owner, repo, number), ai, activePrCache, regenerate: true, ct));

    return app;
}

// Shared gate chain (spec §7): D111 IsSubscribed → seam resolve (Off/Preview/Live gating lives in
// Resolve) → Summarize/Regenerate. Provider failure → 503 (never 500). regenerate=true forces a
// fresh, IsRetry-tagged re-spend.
internal static async Task<IResult> ResolveSummaryAsync(
    PrReference prRef, IAiSeamSelector ai, IActivePrCache activePrCache, bool regenerate, CancellationToken ct)
{
    if (!activePrCache.IsSubscribed(prRef))
        return Results.NoContent();

    var summarizer = ai.Resolve<IPrSummarizer>();
    try
    {
        var summary = regenerate
            ? await summarizer.RegenerateAsync(prRef, ct).ConfigureAwait(false)
            : await summarizer.SummarizeAsync(prRef, ct).ConfigureAwait(false);
        return summary is null ? Results.NoContent() : Results.Ok(summary);
    }
    catch (LlmProviderException)
    {
        return Results.StatusCode(StatusCodes.Status503ServiceUnavailable);
    }
}
```

```ts
import { useCallback, useEffect, useRef, useState } from 'react';
import { getAiSummaryResult, regenerateAiSummary } from '../api/aiSummary';
import type { PrReference, PrSummary } from '../api/types';

export interface AiSummaryState {
  summary: PrSummary | null;
  loading: boolean;
  error: boolean;
  isStale: boolean;
  regenerating: boolean;
  regenerateError: boolean;
  regenerate: () => Promise<void>;
}

export function useAiSummary(
  prRef: PrReference,
  enabled: boolean,
  subscribed: boolean,
  baseShaChanged: boolean,
): AiSummaryState {
  const [summary, setSummary] = useState<PrSummary | null>(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(false);
  const [regenerating, setRegenerating] = useState(false);
  const [regenerateError, setRegenerateError] = useState(false);
  // isStale = baseShaChanged && !staleCleared. A fresh fetch/regenerate clears staleness even
  // while the parent's baseShaChanged latch stays true; a rising base-change edge re-stales.
  const [staleCleared, setStaleCleared] = useState(false);
  // Synchronous in-flight guard. A ref (not the `regenerating` state) closes the double-click
  // double-spend window that a state-based check leaves open under React 18 batching — two clicks
  // arriving before the first setState commit would both pass a state guard.
  const inFlight = useRef(false);

  useEffect(() => {
    if (!enabled || !subscribed) {
      setSummary(null);
      setLoading(false);
      setError(false);
      return;
    }
    let cancelled = false;
    setSummary(null);
    setLoading(true);
    setError(false);
    getAiSummaryResult(prRef).then((r) => {
      if (cancelled) return;
      if (r.kind === 'ok') {
        setSummary(r.summary);
        setLoading(false);
        setError(false);
        setStaleCleared(true); // a freshly fetched summary is current by construction
      } else if (r.kind === 'error') {
        setSummary(null);
        setLoading(false);
        setError(true);
      } else {
        setSummary(null);
        setLoading(false);
        setError(false);
      }
    });
    return () => {
      cancelled = true;
    };
    // baseShaChanged is INTENTIONALLY omitted from deps: a base move must not auto-refetch (token
    // discipline). regenerate() is the only re-fetch trigger.
    // eslint-disable-next-line react-hooks/exhaustive-deps -- stable primitive prRef fields; baseShaChanged omitted by design (#374)
  }, [prRef.owner, prRef.repo, prRef.number, enabled, subscribed]);

  // A rising base-change edge re-stales the card (un-clears the local override).
  useEffect(() => {
    if (baseShaChanged) setStaleCleared(false);
  }, [baseShaChanged]);

  const regenerate = useCallback(async () => {
    if (!enabled || !subscribed || inFlight.current) return; // ref guard is synchronous
    inFlight.current = true;
    setRegenerating(true);
    setRegenerateError(false); // "transient" = cleared on the next deliberate attempt (§9)
    try {
      const r = await regenerateAiSummary(prRef);
      if (r.kind === 'ok') {
        setSummary(r.summary);
        setStaleCleared(true);
      } else if (r.kind === 'error') {
        setRegenerateError(true); // retain the present body (§9)
      }
    } finally {
      inFlight.current = false;
      setRegenerating(false);
    }
    // eslint-disable-next-line react-hooks/exhaustive-deps -- stable primitive prRef fields (#331/#374)
  }, [prRef.owner, prRef.repo, prRef.number, enabled, subscribed]);

  const isStale = baseShaChanged && !staleCleared;

  return { summary, loading, error, isStale, regenerating, regenerateError, regenerate };
}
```

> **Notes for the implementer.**
> - **`regenerateError` is "transient" in the §9 sense of *cleared on the next deliberate Regenerate*** (the `setRegenerateError(false)` at the top of `regenerate()`), NOT auto-dismissed on a timer. The tests assert presence-until-retry; do not add a `setTimeout` dismiss.
> - **The `inFlight` ref — not the `regenerating` state — is what makes the double-spend guard synchronous** (§8 "disables immediately"). The card's `disabled={regenerating}` is the visual half; the ref is the correctness half. (ce-doc-review design-lens.)
> - **Known bound — same-PR double rebase (tracked in #437, DECISION PENDING).** Because the parent latches `baseShaChanged` as a *boolean*, a SECOND base move after a Regenerate produces no false→true edge, so the rising-edge effect does not re-stale and the chip stays hidden until PR navigation / head-change. This is a *reachable* state (regenerate → base moves again → stale summary shown as fresh, no affordance), flagged by ce-doc-review (design-lens P2 + adversarial). The robust fix is a **monotonic base-change counter** in `useActivePrUpdates` (every increment re-stales), threaded as `baseShaChangedCount: number` through this hook in place of the boolean — see **#437**. This slice ships the boolean to match spec §8; **the human plan review decides whether to pull #437's counter into P1b-1 or ship the boolean with this documented bound.** If pulled in: change Task 12 to expose `baseShaChangedCount`, this hook's 4th param to the count, the `staleCleared` pair to `lastClearedCount` (`isStale = count > lastClearedCount`; set `lastClearedCount = count` on each fresh fetch/regenerate), and Task 15 to thread the count.
